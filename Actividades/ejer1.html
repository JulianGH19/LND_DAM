    Internet IPv6

    Tema 1.-Introducción

    IPv6 (Internet Protocol version 6) es la evolución de IPv4, el protocolo que permite identificar y enrutar dispositivos en Internet. Su motivación principal fue resolver el agotamiento de direcciones IPv4: mientras IPv4 usa 32 bits (unos 4.300 millones de direcciones), IPv6 usa 128 bits, ofreciendo una cantidad enorme de direcciones y facilitando el crecimiento de redes, servicios y dispositivos conectados. Además de ampliar el espacio de direcciones, IPv6 simplifica ciertos aspectos del encaminamiento y está diseñado para convivir con IPv4 durante una larga etapa de transición.

    Una dirección IPv6 se escribe en hexadecimal, en ocho grupos (hextetos) separados por dos puntos, por ejemplo: 2001:0db8:0000:0000:0000:0000:0000:0001. Para hacerla más legible se permiten abreviaciones: los ceros a la izquierda de cada grupo se pueden omitir, y una secuencia larga de grupos “0000” se puede comprimir una sola vez usando “::”. También existen direcciones especiales, como la de loopback (::1), la no especificada (::) o las link-local (que suelen empezar por fe80::) usadas dentro del enlace local. A nivel práctico, IPv6 evita el broadcast y emplea multicast y anycast para varios tipos de comunicación en red.

    En IPv6 es habitual la autoconfiguración (SLAAC), donde un equipo puede obtener una dirección sin necesidad de un servidor DHCP tradicional, usando anuncios de router (Router Advertisements). Aun así, también existe DHCPv6 para entornos que lo prefieren. IPv6 incorpora soporte obligatorio de IPsec a nivel de estándar (aunque su uso real depende del despliegue) y usa ICMPv6 para funciones clave como descubrimiento de vecinos (Neighbor Discovery). Como la transición desde IPv4 no es inmediata, se utilizan mecanismos como doble pila (dual stack), túneles o traducción entre protocolos según el caso. En conjunto, IPv6 es una base moderna para redes.
